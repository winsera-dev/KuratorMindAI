# Plan 04-01: Testing Infrastructure & Data Baseline

**Focus:** Setup, Seeding, and OCR Benchmarking (Module 4)
**Autonomous:** YES
**Wave:** 1 (Foundation)

## 🎯 Objective
Initialize the testing infrastructure and baseline data quality measurements. This plan sets up the environment and implements OCR benchmarking to ensure data ingestion integrity.

**Why this matters:** High-stakes legal data requires a "Golden Dataset" baseline to prevent hallucination during ingestion (Module 4).

## 🛠️ Technical Stack & Context
- **Backend:** Pytest, JiWER (Metrics), Supabase Admin
- **Frontend:** Playwright (E2E), Next.js 15
- **Reference:** `e2e_testing_plan.md` Modules 1, 2, and 4.

---

## 📝 Implementation Steps

### Task 1: Testing Environment & Tooling Setup (M1, M2)
Configure core testing frameworks and setup initial authentication state mocks.

- **Files:** `apps/agents/pyproject.toml`, `apps/web/package.json`, `apps/web/playwright.config.ts`
- **Action:**
    - Add test dependencies: `jiwer`, `splink`, `pytest-asyncio`, `pytest-env` (Backend).
    - Install `@playwright/test` and initialize config (Frontend).
    - **M1/M2 Setup:** Configure Playwright global setup to handle Supabase Auth session persistence (TC-AUTH-02).
- **Verification:** 
    - `pytest --version` returns 8.x
    - `npx playwright --version` returns current version

### Task 2: Database Seeding Engine & Supabase Fixtures
Create a deterministic database reset mechanism for "Clean State" testing.

- **Files:** `apps/agents/tests/conftest.py`, `tests/scripts/seed_test_data.py`
- **Action:**
    - Implement `seed_test_data.py` to:
        1. Clear `audit_flags`, `claims`, and `case_documents`.
        2. Seed the **Sritex** and **Mitralanggeng** cases with correct UUIDs.
        3. Setup a "User B" for cross-tenant isolation testing (TC-SEC-02).
    - Use `SUPABASE_SERVICE_ROLE_KEY` to bypass RLS during seeding.
- **Verification:** 
    - `python tests/scripts/seed_test_data.py` resets DB to known state.

### Task 3: OCR Benchmark Suite (JiWER) (M4)
Implement quantitative measurement of character/word error rates for legal PDFs.

- **Files:** `tests/scripts/ocr_benchmark.py`
- **Action:**
    - Use `jiwer` to compare agent-extracted text against `tests/data/test_sritex_claims.csv` (Ground Truth).
    - **TC-DOC-01/02:** Validate extraction success for both PDF and Excel.
- **Verification:** 
    - `python tests/scripts/ocr_benchmark.py` outputs CER/WER.
    - **Target:** CER < 5% for high-quality documents.
