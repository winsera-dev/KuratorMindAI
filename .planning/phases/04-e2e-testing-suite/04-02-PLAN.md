# Plan 04-02: Backend API & Domain Logic Verification

**Focus:** API Integrity (Module 9) and Forensic Logic (Modules 6, 7)
**Autonomous:** YES

## 🎯 Objective
Verify the core functional logic of the KuratorMind AI backend. This plan focuses on ensuring API routes are secure and that legal/forensic agents accurately interpret Indonesian bankruptcy law (UU 37/2004).

**Why this matters:** Platform credibility depends on 100% accurate financial discrepancy detection and strict RLS enforcement (Module 10).

- **Backend:** Pytest, FastAPI, Supabase Python SDK
- **Domain:** Indonesian Bankruptcy Law (UU 37/2004), Forensic Accounting
- **Reference:** `e2e_testing_plan.md` Modules 6, 7, 9, 10.

    - **TC-DISC-02:** Verify `CrossCaseTab` logic (Probabilistic entity resolution for "Heryanto").
- **Verification:** 
    - `pytest apps/agents/tests/test_compliance.py`
    - `pytest apps/agents/tests/test_audit_logic.py`

### Task 3: Security & RLS Validation (Module 10)
Directly test Supabase RLS policies from the Python backend.

- **Files:** `apps/agents/tests/test_security.py`
- **Action:**
    - **TC-SEC-02:** Use a standard user token to attempt a `SELECT` on another user's `case_id`. Assert empty response.
- **Verification:** 
    - `pytest apps/agents/tests/test_security.py`
