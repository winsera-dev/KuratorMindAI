# Plan 04-03: Frontend E2E & Detailed Experience

**Focus:** User Journeys (M1-M3), AI Experience (M5), and UX Polish (M11)
**Autonomous:** YES
**Wave:** 3 (Experience)

## 🎯 Objective
Verify the end-to-end user journeys of the KuratorMind AI platform using **Playwright**. This plan ensures that the reactive UI correctly visualizes multi-agent orchestration and maintains strict security boundaries.

**Why this matters:** User confidence depends on a polished "Detail Experience" (Module 11) where AI reasoning is transparent, cited, and secure.

## 🛠️ Technical Stack & Context
- **Frontend:** Next.js 15, Tailwind, Lucide Icons
- **E2E:** Playwright (Chromium)
- **Reference:** `e2e_testing_plan.md` Modules 1, 2, 3, 5, 8, 11, 12.

---

## 📝 Implementation Steps

### Task 1: Auth, Dashboard & Case CRUD (Modules 1, 2, 3)
Validate core workflow from login to case creation.

- **Files:** `apps/web/e2e/auth_dashboard.spec.ts`, `apps/web/e2e/case_management.spec.ts`
- **Action:**
    - **TC-AUTH-01/02:** Test register/login redirect flows.
    - **TC-DASH-01:** Verify **shimmer skeletons** appear during load (Module 11).
    - **TC-CASE-01 to 07:** Test the full CRUD cycle for a case, including the auto-update of `stage_started_at`.
- **Verification:** 
    - `npx playwright test apps/web/e2e/auth_dashboard.spec.ts`

### Task 2: AI Chat & Forensic Experience (Module 5)
Verify real-time SSE streaming and agent routing.

- **Files:** `apps/web/e2e/ai_chat_forensics.spec.ts`
- **Action:**
    - **TC-CHAT-01/02:** Send query, verify word-by-word streaming and **citation highlights**.
    - **TC-CHAT-05 to 08:** Verify Orchestrator correctly routes to specialist agents (routing labels in UI).
    - **TC-CLAIM-04/05:** Verify **AuditFlagCard** appears with forensic reasoning.
- **Verification:** 
    - `npx playwright test apps/web/e2e/ai_chat_forensics.spec.ts`

### Task 3: Report Generation & UX Polish (Modules 8, 11, 12)
Validate final outputs and platform performance.

- **Files:** `apps/web/e2e/report_gen.spec.ts`
- **Action:**
    - **TC-RPT-01 to 05:** Trigger report, verify progress state, and download resulting PDF.
    - **TC-DASH-06:** Throttle network to 3G and verify skeleton stability (Module 12).
- **Verification:** 
    - `npx playwright test apps/web/e2e/report_gen.spec.ts`
