# Plan 04-04: Advanced Forensic Intelligence & Stability

**Focus:** Completing the "Closing the Loop" mandate from research.
**Autonomous:** YES
**Wave:** 4 (Intelligence & Stability)

## 🎯 Objective
Fully implement the advanced forensic and stability requirements identified in the research phase. This plan elevates the system from basic logic to high-fidelity forensic intelligence, ensuring legal standing verification, fraud detection (Benford's Law), and connection stability.

**Why this matters:** Court-ready forensic evidence requires not just reading data, but identifying patterns of manipulation and strictly adhering to "Simple Proof" legal standards.

## 🛠️ Technical Stack & Context
- **Forensic Math:** Benford's Law (First-digit analysis)
- **Legal Standards:** UU 37/2004 "Simple Proof" (*Pembuktian Sederhana*)
- **Infrastructure:** SSE Heartbeats, Semantic Evals (Vector Embeddings)
- **Reference:** `.planning/research/PITFALLS.md`, `.planning/research/ARCHITECTURE.md`

## 📝 Implementation Steps

### Task 1: Forensic Deep Scan (Benford's Law)
Implement statistical outlier detection to identify potentially manipulated or "cooked" creditor lists.

- **Files:** `apps/agents/kuratormind/services/accounting.py`
- **Action:**
    - Implement `perform_benford_analysis(amounts: list[float])`.
    - Calculate the distribution of the first significant digits (1-9).
    - Compare against Benford’s theoretical distribution.
    - Flag cases where the Chi-squared deviation is > critical threshold.
- **Verification:** 
    - Test with a list of random digits (should fail/flag).
    - Test with a natural financial dataset (should pass).

### Task 2: "Simple Proof" Evidence Gate
Ensure that bankruptcy eligibility checks verify the presence of mandatory legal standing documents.

- **Files:** `apps/agents/kuratormind/services/compliance.py`
- **Action:**
    - Update `check_bankruptcy_eligibility`.
    - Add a `meets_simple_proof` check.
    - Assert that for every critical claim, there is a reference to a "Contract", "Invoice", "Agreement", or "Akta" in the supporting metadata.
- **Verification:** 
    - Eligibility should return `False` if no source document types are linked to the claims.

### Task 3: SSE Heartbeat Protocol
Prevent load balancers and browsers from dropping the connection during deep agent reasoning.

- **Files:** `apps/agents/kuratormind/api/routes/chat.py`
- **Action:**
    - Implement a background heartbeat task within the `generate()` generator.
    - Yield a `: heartbeat` comment event every 15 seconds if no tokens have been sent.
- **Verification:** 
    - Monitor network tab in dev tools during a long chat query; verify `: heartbeat` messages appear in the raw stream.

### Task 4: Semantic Reasoning Evaluation Suite
Implement "LLM-as-Judge" style tests using vector similarity instead of exact matching.

- **Files:** `apps/agents/tests/test_reasoning_quality.py`
- **Action:**
    - Create a test suite that sends 5 critical legal questions to the `regulatory_scholar`.
    - Embed the agent's response and compare it against a "Golden Answer" embedding.
    - Assert `cosine_similarity > 0.85`.
- **Verification:** 
    - `pytest apps/agents/tests/test_reasoning_quality.py` passes.
